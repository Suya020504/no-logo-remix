# NO LOGO REMIX UI·프론트엔드 기능명세서 v1.0

## 0. 문서 목적

이 문서는 **기획자 → UI 디자이너 → 프론트엔드 개발자**에게 전달하는 작업 기준서다. 목표는 NO LOGO REMIX의 핵심 경험인 아래 흐름을 모바일 웹에서 끊김 없이 보여주는 것이다.

> 관심 문제 선택 → 블라인드 기술 발견 → 기업·부스 공개 → 부스 방문 인증 → 기술 토큰 획득 → 2~3개 기술 조합 → AI 리믹스 → 아이디어 공개·투표·공유 → 참여 데이터 축적

핵심 메시지:

> **이름을 보고 기술을 고르는 전시에서, 기술을 먼저 고르고 기업을 발견하는 전시로.**

---

# 1. 제품 구조

## 1-1. 서비스 위계

- **NO LOGO REMIX**: 관람객이 경험하는 메인 참여 프로그램
- **REMIX AI KIT**: 기업 승인 기술정보, 미션, 프롬프트, 출력규격을 묶은 운영 엔진
- **MIXI(가칭)**: 관람객을 안내하고 사전·현장·사후 콘텐츠를 연결하는 AI 마스코트

## 1-2. 사용자

### 2030 관람객

- 브랜드 인지도보다 기술의 문제해결 가능성을 보고 선택한다.
- 실제 부스를 방문해 기술을 이해하고 토큰을 획득한다.
- 서로 다른 기업 기술을 조합해 개인화된 아이디어를 만든다.
- 결과물을 전광판과 SNS에 공유하고 투표한다.

### 참가기업

- 블라인드 선택 수, 부스 방문 전환율, 기술 조합 빈도, 공유 반응을 확인한다.
- 개인정보는 별도 동의가 있는 경우에만 후속 연락에 활용한다.

### 전시 주최사

- 중앙 공용 체험을 운영하고 여러 부스로 이동을 유도한다.
- 전시별 기술카드·프롬프트·데이터 자산을 축적한다.

# 2. 플랫폼과 화면 규격

## 2-1. 관람객용 모바일 웹

- 기준 프레임: **390 × 844px**
- 최소 지원 폭: 360px
- 모바일 우선 설계
- 로그인 강제 금지: 익명 세션으로 시작하고, 결과 저장·기업 연락 동의 단계에서만 선택 입력
- 하단 고정 CTA와 한 손 조작을 우선

## 2-2. 현장 전광판

- 기준 프레임: **1920 × 1080px**
- 5~8m 거리에서도 읽히도록 큰 제목과 최소 정보만 표시
- 최신작, 인기작, 투표 QR, 현재 참여 수를 자동 순환

## 2-3. 기업·주최사용 대시보드

- 기준 프레임: **1440 × 1024px**
- 심사용 프로토타입에서는 정적 가상 데이터 사용 가능
- 모든 수치에 `가상 예시` 또는 `파일럿 측정 항목` 표시

# 5. 페이지별 기능명세

## P01. 시작·온보딩

### 목표

사용자가 5초 안에 서비스의 핵심을 이해하고 체험을 시작하도록 한다.

### 핵심 문구

- 타이틀: **브랜드는 숨기고, 기술을 발견하세요.**
- 서브: `관심 기술을 고르고 실제 부스를 방문한 뒤, 여러 기업의 기술로 나만의 미래 아이디어를 만들어보세요.`
- CTA: **QR로 시작하기** 또는 **체험 시작하기**

### 화면 구성

1. NO LOGO REMIX 로고·행사명
2. MIXI 환영 일러스트
3. 5단계 미니 플로우
   - 발견 → 공개 → 방문 → 리믹스 → 공유
4. 예상 체험시간 `약 10~15분`
5. Primary CTA
6. 개인정보 최소수집 안내 링크

### 기능

- 익명 세션 생성
- 기존 세션이 있으면 `이어서 하기` 표시
- 행사장 QR에서 진입한 경우 전시 ID 자동 설정

### 상태

- 정상
- 세션 복구
- 행사 종료
- 네트워크 오류

### 디자인 포인트

- 첫 화면은 흰 배경과 보라색 포인트로 가볍게 구성
- MIXI는 화면의 25% 이하로 사용해 서비스 정보보다 튀지 않게 함
- 핵심 CTA는 화면 하단 엄지 영역에 배치

## P02. 관심 문제·미션 선택

### 목표

관람객이 기업·기술명보다 자신의 관심과 생활문제를 먼저 생각하게 한다.

### 핵심 문구

- 타이틀: **어떤 문제를 해결하고 싶나요?**
- 보조: `하나의 메인 미션을 고르면 관련 기술을 추천해드려요.`

### 추천 미션

- 폭염 속 자취방 전기료 줄이기
- 플라스틱 없는 뷰티 루틴 만들기
- 출퇴근 시간을 줄이는 이동 서비스
- 1인 가구 식품 낭비 줄이기
- 정전에도 작동하는 미래도시 만들기
- 나만의 문제 직접 입력

### 화면 구성

1. 카테고리 칩: 환경·에너지 / 건강·의료 / 모빌리티 / 식품 / 스마트시티 / 기타
2. 미션 카드 6개
3. 선택 이유 한 줄 입력(선택)
4. 관심 분야 최대 3개 칩
5. CTA: **기술 발견하러 가기**

### 기능

- 메인 미션 1개 필수 선택
- 관심 분야 최대 3개 선택
- 선택값을 이후 기술 추천과 AI 프롬프트에 사용

### 상태

- 미선택: CTA 비활성
- 선택 완료: 카드 테두리·체크 표시
- 직접 입력: 40자 제한

### 디자인 포인트

- 문제 중심의 일러스트 카드 사용
- 기업·제품 이미지는 노출하지 않음
- 선택 완료 시 과한 색 변화보다 체크·테두리·배경색을 함께 사용

## P03. 블라인드 기술카드

### 목표

기업명, 로고, 기업 규모를 가린 상태에서 기술의 기능과 해결문제만 보고 선택하게 한다.

### 핵심 문구

- 타이틀: **이름 대신 기술을 보고 골라보세요.**
- 보조: `선택 전까지 기업명은 공개되지 않습니다.`

### 카드에 노출할 정보

- 기술 번호: `TECH No.12`
- 해결하는 문제
- 핵심 기능 1문장
- 적용 분야 태그 2~3개
- 활용 가능성 아이콘
- `기업명 비공개` 배지

### 카드에서 숨길 정보

- 기업명·로고
- 기업 규모
- 부스 크기·사진
- 유명 브랜드를 추측할 수 있는 고유 제품명

### 화면 구성

1. 추천순 / 새로운 기술 / 인기순 필터
2. 기술카드 2열 그리드 또는 가로 스와이프
3. 카드 탭 시 상세 바텀시트
4. 선택한 카드 하단 고정 요약
5. CTA: **이 기술 선택하기**

### 기능

- 카드 순서는 세션별 무작위화 가능
- 상세 바텀시트에서 `해결 문제 / 기능 / 활용 예시` 확인
- 1차 선택 후 확인 모달
- 선택 이유 선택: 흥미로움 / 내 문제와 관련 / 새로움 / 활용 가능성

### 상태

- 로딩 스켈레톤
- 데이터 없음
- 선택 완료
- 이미 선택한 기술

### 디자인 포인트

- 블라인드 단계는 딥 네이비 카드와 미스터리 그래픽 사용
- 카드 뒷면·물음표·큐브 모티프 활용
- AI 느낌의 과도한 네온 대신 산업기술의 정교함을 살림

## P04. 기업 공개·부스 안내

### 목표

블라인드 선택에서 생긴 호기심을 실제 기업 발견과 부스 이동으로 전환한다.

### 핵심 문구

- 타이틀: **당신이 고른 기술의 주인공을 공개합니다.**
- 서브: `이 기술은 A기업의 저전력 센서 기술입니다.`

### 화면 구성

1. 카드 플립 또는 커튼 오픈 리빌 모션
2. 기업명·로고
3. 기술명과 쉬운 설명
4. 개발 배경 2~3문장
5. 부스 위치: `B홀 B-12`
6. 미니맵·예상 이동시간
7. 부스에서 확인할 미션 1개
8. CTA: **이 부스로 가기**

### 기능

- 지도 보기
- 길찾기 안내
- 기술 저장
- 부스 방문 전에는 토큰이 잠긴 상태로 표시

### 상태

- 기업 정보 정상
- 지도 정보 없음: 텍스트 위치만 제공
- 부스 운영 종료

### 디자인 포인트

- 블라인드 카드의 어두운 색에서 밝은 기업 카드로 전환
- 기업 로고가 지나치게 화면을 지배하지 않도록 동일 규격 적용
- 리빌 애니메이션은 500ms 이내

## P05. 부스 QR 인증

### 목표

실제 부스 방문을 확인하고 기술 토큰을 활성화한다.

### 핵심 문구

- 타이틀: **부스에 도착했나요?**
- 보조: `부스의 QR을 스캔하고 기술 설명을 확인하세요.`

### 화면 구성

1. 카메라 스캔 영역
2. 플래시·카메라 전환
3. `직원에게 인증코드 받기` 대체 방식
4. 인증 전 체크 항목
   - 기술 설명 듣기
   - 제품·데모 확인하기
5. 도움말

### 기능

- 카메라 권한 요청
- QR 값과 현재 선택 기술 매칭
- 중복 인증 차단
- 잘못된 부스 QR 안내
- 성공 시 기술 토큰 발급

### 상태

- 권한 허용 전
- 스캔 중
- 인증 성공
- 잘못된 QR
- 중복 인증
- 네트워크 오류

### 성공 화면

- 토큰 언락 애니메이션
- `기술 토큰을 획득했어요!`
- 현재 보유 토큰 `1 / 3`
- CTA: **다음 기술 찾기** / **토큰함 보기**

### 디자인 포인트

- 성공 애니메이션 1.5초 이내
- 진동·사운드는 선택적 사용
- QR 인식이 어려운 환경을 위해 숫자 코드 입력 제공

## P06. 나의 기술 토큰

### 목표

수집 상태를 한눈에 보여주고 2~3개 기술을 모으도록 동기를 부여한다.

### 핵심 문구

- 타이틀: **나의 기술 토큰**
- 상태: `리믹스까지 1개의 기술이 더 필요해요.`

### 화면 구성

1. 보유 토큰 슬롯 3개
2. 획득 토큰 카드
   - 기술명
   - 기업명
   - 부스 위치
   - 핵심 역할
3. 잠긴 슬롯
4. 추천 다음 기술
5. 조합 가능 여부 안내
6. CTA: **AI 리믹스 시작하기**

### 기능

- 최소 2개, 최대 3개 선택
- 같은 기업의 기술만 선택된 경우 경고
- 기업 AI 허용등급에 따라 조합 가능 범위 표시
- 토큰 삭제·교체

### 상태

- 토큰 0개
- 1개
- 2개: CTA 활성
- 3개: 최대 수집
- 조합 제한

### 디자인 포인트

- 토큰 카테고리별 보조색 사용
- 색만으로 구분하지 않고 아이콘·텍스트 동시 사용
- 수집 자체가 게임처럼 느껴지되 지나친 게임 UI는 피함

## P07. AI 리믹스 설정·생성

### 목표

AI 사용 경험이 적은 사용자도 버튼 선택만으로 일정한 품질의 결과물을 만들게 한다.

### 핵심 문구

- 타이틀: **기술을 섞어 새로운 아이디어를 만들어보세요.**
- 안내: `빈 프롬프트를 쓰지 않아도 MIXI가 조합을 도와드려요.`

### 입력 단계

1. 기술 2~3개 선택
2. 해결 미션 확인 또는 변경
3. 결과 스타일 선택
   - 현실형
   - 미래형
   - 재미형
4. 아이디어 대상 선택
   - 제품
   - 서비스
   - 공간·도시
5. 생성 실행

### 기능

- 기업 승인정보만 프롬프트 입력값으로 사용
- 금지 표현·미승인 성능·수치 생성 방지
- 생성 진행률 표시
- 목표 생성시간 1분 내외
- 오류 시 준비된 예시 결과로 대체 가능

### 생성 중 화면

- MIXI 리믹스 모드
- 단계 메시지
  - 기술 분석 중
  - 연결점 찾는 중
  - 이미지 만드는 중
  - 결과 정리 중
- 취소·백그라운드 전환 안내

### 상태

- 입력 미완료
- 생성 중
- 생성 지연
- 안전필터 재생성
- 생성 실패

### 디자인 포인트

- 선택형 폼을 한 화면에 몰아넣지 않고 단계형으로 구성
- 생성 중에는 실제 진행률처럼 오해되지 않도록 `예상 단계` 표현 사용
- AI를 주인공으로 보이게 하기보다 선택한 기술카드를 중심에 배치

## P08. 결과 상세

### 목표

생성된 아이디어를 이해하고 저장·게시·공유하도록 한다.

### 출력 규격

- 아이디어명: 15자 내외
- 한 문장 설명: 100자 이내
- 콘셉트 이미지 1장
- 사용 기술 2~3개와 역할
- 참여기업·부스 정보
- SNS 공유문구
- 면책 문구

### 화면 구성

1. 결과 대표 이미지
2. 아이디어명
3. 한 문장 콘셉트
4. `어떤 기술이 어떻게 쓰였나요?` 섹션
5. 기업·부스 태그
6. 결과 스타일 배지
7. 면책 문구
   - `관람객과 AI가 만든 콘셉트 제안이며, 실제 기술성과 상용화 가능성은 별도 검토가 필요합니다.`
8. CTA
   - 아이디어 트리에 게시
   - 저장
   - 공유
   - 한 번 다시 만들기

### 기능

- 결과 저장
- 이미지 다운로드 또는 Web Share API 호출
- 결과 게시 전 공개 동의
- 재생성 1회 제한 가능

### 상태

- 정상 결과
- 이미지 생성 실패: 텍스트 결과 우선 제공
- 금칙어 검토 중
- 게시 완료

### 디자인 포인트

- 결과 카드는 SNS 세로형 9:16으로도 재사용 가능하게 설계
- 기술 역할을 기업 로고만 나열하지 말고 문장으로 설명
- 마스코트는 결과 이미지의 10~15% 이하 보조 요소로 사용

### 핵심 문구

6. SNS 공유 버튼
7. 현장 전광판 노출 여부

### 기능

- 자기 아이디어 투표 허용 여부를 운영정책으로 설정

## P10. 마이페이지·참여 결과

사용자가 자신의 전시 경험을 회고하고 사후 콘텐츠와 기업 연결을 선택하게 한다.

### 화면 구성

   - 생성 아이디어 수
   - 받은 투표 수
3. 획득 토큰
4. 내 아이디어
5. 디지털 배지
6. 기업 자료·채용·상담 연결 동의
7. 개인정보 설정·삭제

### 기능

- 동의 기반 연락처 입력

### 상태

- 익명 사용자

- 연락 동의 완료

### 디자인 포인트

## S01. 현장 라이브 전광판

### 목표

관람객의 결과물을 현장 분위기와 참여 확산을 만드는 콘텐츠로 활용한다.

### 레이아웃

- 좌측 70%: 최신 아이디어 6~8개 그리드
- 우측 상단: 실시간 TOP 3
- 우측 중단: 참여 수·투표 수·생성 수
- 우측 하단: 투표 QR과 짧은 안내
- 하단 티커: 다음 REMIX LIVE 시간

### 기능

- 10~15초 자동 순환
- 최신작·인기작·기업 다양성 균형 노출
- 운영자 승인 후 공개
- 전체화면·오프라인 캐시

### 디자인 포인트

- 텍스트는 제목 1줄, 설명 2줄까지만
- 전광판에서 기업 로고가 과도하게 반복되지 않도록 동일 크기 적용
- 움직임은 카드 페이드·슬라이드 정도로 제한

## S02. 기업 관심 데이터 대시보드

### 목표

일반 관람객 참여가 참가기업의 실질적 인사이트로 이어진다는 점을 보여준다.

### 핵심 KPI

- 블라인드 카드 노출 수
- 블라인드 기술 선택 수
- 기업 공개 후 부스 QR 인증 수
- 부스 방문 전환율
- 다른 기술과의 조합 횟수
- 생성 결과물 수
- 투표·공유 수
- 별도 동의 자료 요청 수

### 화면 구성

1. KPI 카드 4개
2. `노출 → 선택 → 공개 → 방문 → 리믹스 → 공유` 퍼널
3. 인기 기술 조합 TOP 5
4. 관심 문제 분포
5. 시간대별 방문 추이
6. 관람객 속성은 집계 단위로만 표시
7. 데이터 해석 문장

### 주의

- 프로토타입 수치는 반드시 `가상 예시`로 표시
- 실제 구매의향·시장검증으로 단정하지 않고 `2030 관심 신호`로 표현
- 개인정보는 개별값이 아니라 집계값 우선

# 13. 프론트엔드 인수 기준

- 360px 화면에서도 가로 스크롤이 발생하지 않는다.
- 핵심 참여 흐름이 새로고침 후에도 유지된다.
- P03에서 기업명·로고가 Reveal 전에는 DOM 또는 화면에 노출되지 않는다.
- 토큰이 2개 미만이면 리믹스 CTA가 비활성화된다.
- QR 데모 인증 후 해당 기술 토큰이 즉시 반영된다.
- 생성 중·성공·실패 상태가 모두 존재한다.
- 결과카드는 모바일 화면과 9:16 공유 이미지에서 동일한 정보구조를 사용한다.
- 투표 중복 상태가 표시된다.
- 개인정보 제공은 기본값이 꺼진 옵트인이다.
- 키보드 포커스와 스크린리더용 레이블을 제공한다.
- 버튼 터치영역은 최소 44px이다.

# 14. 기획서에 넣을 화면 우선순위

공식 기획서에는 아래 4개 화면만 선별해 넣고, 전체 프로토타입은 QR로 연결한다.

1. 블라인드 기술카드
2. 기업 공개·부스 위치
3. AI 리믹스 결과카드
4. 기업 관심 데이터 대시보드

이 4개만으로도 `공정한 발견 → 실제 부스 이동 → 개인화 공동창작 → 참가기업 실익`이 한 번에 보인다.

# 15. AI·콘텐츠 표시 원칙

- 결과물마다 `AI를 활용한 콘셉트 제안`임을 표시
- 기업이 승인한 공개 정보만 사용
- 확인되지 않은 수치·성능·효과 생성 금지
- 타 브랜드·캐릭터·작가 화풍 모방 금지
- 제출자료에는 생성형 AI 활용 여부와 범위를 별도 명시

권장 표기:

> 생성형 AI 활용 범위: 회의내용 구조화, 문장 교정, 프롬프트 템플릿 설계, 프로토타입 및 마스코트 시안 제작에 활용했으며, 최종 기획과 검토는 팀이 수행함.
