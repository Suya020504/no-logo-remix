# 1. 서비스 개요

## 1-1. 한 줄 정의

NO LOGO REMIX는 **브랜드를 가린 기술을 선택하고, 실제 부스를 방문해 기술 토큰을 모은 뒤, AI로 새로운 아이디어를 만들어 공유하는 참여형 전시 플랫폼**이다.

## 1-2. 사용자 여정 10단계 ↔ 화면 매핑 (기준: 사용자 여정 지도 인포그래픽)

모든 화면은 별첨 **사용자 여정 10단계 지도**를 기준으로 설계한다. 진행 내비게이션의 단계 라벨도 아래 명칭을 그대로 사용한다.

| 여정 단계 | 진행 라벨 | 대응 화면 | 핵심 전환 |
|---|---|---|---|
| ① 관심 분야·해결하고 싶은 문제 선택 | 관심 선택 | P01 → P02 | 개인화 시작 |
| ② 기업명이 가려진 기술카드 선택 | 블라인드 | P03 | 공정한 발견 |
| ③ 기업명과 부스 위치 공개 | 기업 공개 | P04 | 호기심 → 방문 의도 |
| ④ 실제 부스 방문 및 QR 인증 | 부스 인증 | P05 | 온라인 → 오프라인 |
| ⑤ 기술 토큰 획득 | 토큰 획득 | P05 성공 → P06 | 방문의 보상화 |
| ⑥ 서로 다른 기술 2~3개 조합 | 기술 조합 | P06 → P07 입력 | 수집 → 창작 |
| ⑦ AI를 통한 아이디어·이미지 생성 | AI 생성 | P07 → P08 | 창작 완성 |
| ⑧ 아이디어 트리·전광판 게시 | 전광판 | P09 게시, S01 | 개인 → 전체 콘텐츠 |
| ⑨ 현장 투표와 SNS 공유 | 투표·공유 | P09 | 확산·사회적 보상 |
| ⑩ 참여 결과 데이터 분석 | 데이터 | P10, S02 | 데이터 자산화 |

디자인은 이 흐름의 **세 번의 전환 — 발견(블라인드→공개), 방문(공개→QR 인증), 창작(토큰→리믹스)** 이 가장 강하게 보이도록 설계한다.

## 1-3. 서비스 위계

- **NO LOGO REMIX**: 관람객이 경험하는 메인 참여 프로그램
- **REMIX AI KIT**: 기업 승인 기술정보·미션·프롬프트·출력규격을 묶은 운영 엔진 (서버)
- **MIXI**: 관람객을 안내하는 AI 마스코트 (흰색+보라 로봇 캐릭터, "안녕! 나는 MIXI야!")

## 1-4. 사용자

| 사용자 | 핵심 행동 |
|---|---|
| 2030 관람객 | 브랜드가 아닌 문제해결력으로 기술 선택 → 부스 방문 → 토큰 수집 → AI 리믹스 → 공유·투표 |
| 참가기업 | 블라인드 선택 수, 부스 방문 전환율, 조합 빈도, 공유 반응 확인 (동의 기반 후속 연락) |
| 전시 주최사 | 중앙 체험 운영, 부스 이동 유도, 전시별 데이터 자산 축적 |

# 4. 공통 내비게이션·UI 규칙

## 4-1. 상단 영역

- 뒤로가기 버튼
- 현재 단계명 (예: `03 블라인드 기술카드`)
- `3 / 10` 형식 진행상태 표시 — 단계 라벨은 여정 지도 명칭(`관심 선택 → 블라인드 → 기업 공개 → 부스 인증 → 토큰 획득 → 기술 조합 → AI 생성 → 전광판 → 투표·공유 → 데이터`)을 따름
- 도움말 아이콘

## 4-2. 하단 내비게이션 (5탭 고정)

1. 홈
2. 기술카드
3. 토큰함
4. 리믹스
5. 마이페이지

> 단, **P01~P05 첫 참여 흐름에서는 하단 탭을 숨기거나 단순화**해 이탈을 줄인다. 탭은 토큰 1개 이상 획득 후 노출 권장.

## 4-3. CTA 위계

- **Primary**: 다음 단계로 이동하는 핵심 행동 (보라색 풀 버튼, 화면 하단 엄지 영역 고정)
- **Secondary**: 저장, 미리보기, 다시 선택 (아웃라인/고스트)
- **Tertiary**: 건너뛰기, 도움말, 취소 (텍스트 링크)
- 비활성 상태에는 **이유를 즉시 표시** (예: "리믹스까지 1개의 기술이 더 필요해요.")

## P01. 시작·온보딩 `/`

### 목적
사용자가 **5초 안에** 프로그램의 차별점을 이해하고 체험을 시작하게 한다.

### 핵심 문구
- 타이틀: **브랜드는 숨기고, 기술을 발견하세요.**
- 서브: `관심 기술을 고르고 실제 부스를 방문한 뒤, 여러 기업의 기술로 나만의 미래 아이디어를 만들어보세요.`
- CTA: **QR 스캔으로 시작하기** / 보조: **닉네임으로 시작하기**

### 화면 구성
1. NO LOGO REMIX 로고·행사명
2. MIXI 환영 일러스트 (화면의 25% 이하)
3. 5단계 미니 플로우: `발견 → 공개 → 방문 → 리믹스 → 공유`
4. 예상 체험시간 `약 10~15분`
5. Primary CTA + 보조 CTA
6. 하단 캡션: `로그인하면 참여 기록이 저장돼요!` + 개인정보 최소수집 안내 링크

### 기능
- 익명 세션 생성 (로그인 없음)
- 기존 참여 이력이 있으면 **이어서 하기** 표시
- 행사장 QR 진입 시 전시 ID 자동 설정
- 간단한 닉네임 설정 (선택)

### 상태
정상 / 세션 복구(이어서 하기) / 행사 종료 / 네트워크 오류

### 디자인 방향
- 흰색 배경 + 보라색 포인트로 가볍게 구성
- MIXI는 화면을 장악하는 캐릭터가 아니라 **참여방법을 설명하는 안내자**로 배치
- 핵심 CTA는 하단 엄지 영역

## P02. 관심 문제·미션 선택 `/mission`

### 목적
기업·제품보다 **관람객이 해결하고 싶은 생활문제를 먼저** 선택하게 한다.

### 핵심 문구
- 타이틀: **어떤 문제를 해결하고 싶나요?**
- 보조: `하나의 미션을 선택하면 관련 기술을 추천해드려요.`
- CTA: **기술 발견하러 가기**

### 미션 카드 (6개)
1. 폭염 속 자취방 전기료 줄이기
2. 1인 가구 식품 낭비 줄이기
3. 플라스틱 없는 뷰티 루틴 만들기
4. 출퇴근 시간을 줄이는 이동 서비스
5. 정전에도 작동하는 미래도시 만들기
6. 직접 문제 입력하기 (자유 입력)

### 화면 구성
1. 관심 분야 버튼 6종 (아이콘 + 라벨, 여정 지도 기준): `환경·에너지 / 일상·생활 / 건강·의료 / 모빌리티 / 식음료 / 미래도시`
2. 미션 카드 6개 (생활상황 일러스트 + 제목 + 설명 1줄 + 선택 체크)
3. 선택 이유 한 줄 입력 (선택 항목)
4. 관심 분야 칩 — 최대 3개
5. Primary CTA

### 기능·규칙
- **메인 미션 1개 필수** 선택 (해결하고 싶은 문제 또는 방문 목적)
- 관심 분야 1~3개 선택
- 직접 입력은 **40자 이내**
- 선택값은 이후 기술 추천과 AI 결과 생성 프롬프트에 사용
- 선택 완료 시 **추천받을 기술카드 수와 예상 체험 시간대** 안내 표시
- 시스템: 선택값 기반 기술카드 우선 추천, 특정 기업 과다 노출 방지를 위한 카드 순서 무작위화, 익명 세션 참여 허용

### 상태
- 미선택: CTA 비활성 (이유 표시)
- 선택 완료: 카드 테두리 + 체크 표시 + 배경색 변화
- 직접 입력: 글자수 카운터 표시

### 디자인 방향
- 제품 사진이 아닌 **생활상황 일러스트** 사용
- 이 단계에서 기업명·브랜드 요소를 일절 노출하지 않음

## P03. 블라인드 기술카드 `/blind-tech`

### 목적
기업 인지도가 아닌 **기술의 문제해결력과 흥미**를 기준으로 선택하게 한다.

### 핵심 문구
- 타이틀: **이름 대신 기술을 보고 골라보세요.**
- 보조: `선택하기 전까지 기업명은 공개되지 않습니다.`
- CTA: **이 기술 선택하기**

### 카드에 노출할 정보
| 항목 | 예시 |
|---|---|
| 기술 번호 | `BLIND TECH CARD No.12 / 20` (전체 카드 수 함께 표기) |
| 해결하는 문제 | `실내 온도 변화 실시간 감지` |
| 핵심 기능 1문장 | `저전력 센서로 온·습도 모니터링` |
| 적용 분야 태그 2~3개 | `스마트홈` `HVAC` `에너지 관리` |
| 활용 가능성 아이콘 | 일상 활용도 표시 |
| 배지 | `기업명 비공개` |

### 카드에서 반드시 숨길 정보
- 기업명·로고
- 기업 규모
- 부스 사진·크기
- 유명 브랜드를 추측할 수 있는 고유 제품명

### 화면 구성
1. 필터: `추천순 / 새로운 기술 / 인기순 / 관심분야` (+ 검색)
2. 기술카드 **2열 그리드** 또는 가로 스와이프
3. 카드 탭 → 상세 **바텀시트** (`해결 문제 / 핵심 기능 / 활용 예시`)
4. 선택한 카드 하단 고정 요약 바 (`선택한 기술 0 / 3`)
5. Primary CTA

### 기능·규칙
- 카드는 **무작위 순서 노출 + 동일한 시각 규격** 적용 (중소기업과 대기업이 기술 자체로 첫 평가를 받도록 설계)
- 최대 3개까지 선택 가능
- 1차 선택 후 확인 모달
- 선택 이유 기록(선택): **`필요성 / 신기함 / 활용성`** 3분류
- 수집 데이터: 브랜드 공개 전 순수 기술 선택 수, **카드별 체류시간**, 선택 이유

### 상태
로딩 스켈레톤 / 데이터 없음 / 선택 완료 / 이미 선택한 기술

### 디자인 방향
- 블라인드 구간은 **딥 네이비 카드 + 큐브·물음표·잠금 모티프**
- 전형적인 네온 AI 화면이 아니라 **산업기술의 정교함과 미스터리감** 강조

## P04. 기업명·부스 위치 공개 `/reveal/:techId`

### 목적
블라인드 선택에서 발생한 호기심을 **실제 기업 발견과 부스 이동**으로 전환한다.

### 핵심 문구
- 타이틀: **당신이 고른 기술의 주인공을 공개합니다.**

5. 활동 기록 타임라인 (예: `14:20 에너랩스 (B-12) 방문 인증`)
6. 기업 관심 데이터 리포트 요약 + `리포트 받기`
7. 관심기업 저장 / 자료 요청·채용·상담 연결
8. 개인정보 설정·데이터 삭제

### 기능·규칙
- 결과물 재다운로드
- 관심기업 저장
- **연락처 제공은 기본 OFF 옵트인** — 사용자가 직접 동의해야 활성화
- 만족도 조사
- 세션 데이터 삭제

### 상태
익명 사용자 / 저장 정보 없음 / 연락 동의 완료 / 행사 종료 후 아카이브 모드

### 개인정보 원칙
일반 참여는 익명으로 진행. 기업에 연락처·전공·관심 직무를 전달하는 기능은 **사용자가 직접 동의해야만 활성화**된다.

### 디자인 방향
- 데이터 과시가 아닌 **개인 경험 회고** 중심
- 다크 배경 카드로 마무리 화면의 특별함 연출 가능 (시안 참조)

## S01. 현장 라이브 전광판 `/live` (1920×1080)

### 목적
관람객의 결과물을 **현장 분위기와 참여 확산**을 만드는 콘텐츠로 활용한다.

### 레이아웃
- 좌측 70%: 최신 아이디어 **6~8개 그리드**
- 우측 상단: 실시간 TOP 3 (LIVE REMIX ARENA)
- 우측 중단: 현재 생성 아이디어 수 · 현장 투표 수 · 참여 수
- 우측 하단: 참여용 QR + 짧은 안내
- 하단 티커: 다음 REMIX LIVE 시간 · 행사 공식 해시태그

### 기능·운영 원칙
- **10~15초 단위 자동 순환**
- 운영자 승인 후 게시
- 최신작·인기작·**기업 다양성**을 함께 고려한 노출 균형
- 전체화면·오프라인 캐시 지원

### 디자인 방향
- 긴 설명 배제: 제목 1줄 + 설명 최대 2줄, **아이디어명·이미지 중심**
- 기업 로고가 과도하게 반복되지 않도록 동일 크기 적용
- 움직임은 카드 페이드·슬라이드 정도로 제한

## S02. 기업 관심 데이터 대시보드 `/dashboard` (1440×1024)

### 목적
일반 관람객 참여가 **참가기업의 실질적 인사이트**로 이어짐을 보여준다.

### 핵심 KPI
1. 블라인드 카드 노출 수
2. 기업 공개 전 기술 선택 수 (예: `1,256회`)
3. 선택 후 실제 부스 방문 수
4. **부스 방문 전환율** (예: `38.7%`)
5. 다른 기업 기술과 조합된 횟수 (예: `842회`)
6. 결과물 생성 수
7. 투표·공유 수
8. 동의 기반 자료 요청 수

### KPI 산식 (대시보드 고정 표기 — 여정 지도 Step 10 확정)

| KPI | 산식 |
|---|---|
| 부스 방문 전환율 | QR 인증자 ÷ 기술 선택자 |
| 공유율 | 유효 SNS 공유자 ÷ 결과물 완성자 |
| 숨은 기업 발견지수 (Hidden Champion 지수) | 중소기업 선택 비중 ÷ 참여 비중 |

### 화면 구성 (권장 시각화)
1. 상단 KPI 카드 4개
2. `노출 → 선택 → 공개 → 방문 → 리믹스 → 공유` **퍼널 차트**
3. 인기 기술 조합 TOP 5
4. 선택한 문제(미션) 분포 — 가로 막대
5. 시간대별 부스 방문 추이
6. 참여자 관심 분야 — 도넛 차트
7. 관람객 만족도 / Hidden Champion 지수
8. 데이터 해석 문장 1~2줄

### 주의 (필수)
- 프로토타입 수치는 반드시 **`가상 예시`** 라고 표기
- 구매 의향·시장 검증으로 표현하지 않고 **`2030 관심 신호`** 로 정의
- 개인정보는 개별값이 아닌 **집계값**만 표시 (`개인정보는 수집하지 않으며, 익명 데이터만 분석 제공됩니다.`)
- **기업별 리포트와 전체 전시 리포트를 분리 제공** — 익명 집계가 기본, 개인정보 전달은 별도 동의
- 인기 미션·기술군은 다음 행사 콘텐츠에 반영, 주최사는 프롬프트·기술데이터·마스코트를 재사용 자산화

# 16. 최종 전달 문장

> **블라인드 기술카드가 관심을 만들고, 기업 공개와 QR 인증이 실제 부스 방문을 만들며, 기술 토큰과 REMIX AI가 공유 가능한 결과물을 만든다. 프론트 디자인은 이 세 번의 전환 — 발견, 방문, 창작 — 이 가장 강하게 보이도록 설계한다.**
