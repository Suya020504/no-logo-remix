# 0. 서비스 한 줄 정의와 여정

**브랜드를 가린 기술을 선택하고, 실제 부스를 방문해 기술 토큰을 모은 뒤, AI로 새로운 아이디어를 만들어 공유하는 참여형 전시 플랫폼.**

사용자 여정 10단계 (진행 라벨):

```
① 관심 선택 → ② 블라인드 → ③ 기업 공개 → ④ 부스 인증 → ⑤ 토큰 획득
→ ⑥ 기술 조합 → ⑦ AI 생성 → ⑧ 전광판 → ⑨ 투표·공유 → ⑩ 데이터
```

화면 매핑: ①=P01·P02, ②=P03, ③=P04, ④·⑤=P05, ⑤·⑥=P06, ⑦=P07·P08, ⑧·⑨=P09(+S01), ⑩=P10(+S02)

# 1. 디자이너 재량 범위 vs 기능 제약

## 1-1. 디자이너 재량 (자유롭게 판단)

- 화면 비율·그리드·레이아웃 구성
- 컬러 팔레트 (기존 보라 계열 안은 **참고용**이며, 확정된 블루 톤 마스코트와의 조화를 포함해 재량으로 결정)
- 타이포그래피, 아이콘 스타일, 일러스트 스타일
- 모션·전환의 표현 방식
- 카드·버튼·컴포넌트의 형태

## 1-2. 기능 제약 (반드시 준수 — 표현 방식은 자유)

| # | 제약 | 이유 |
|---|---|---|
| 1 | **기업 공개(P04) 이전의 모든 화면(P01~P03)에서 기업명·로고·기업 규모·부스 사진·브랜드 추측 가능 제품명을 화면과 DOM 어디에도 노출하지 않는다** — "추측 가능 제품명" 판정은 기술카드 콘텐츠 작성 단계의 편집 체크리스트에서 수행 | 서비스의 핵심 가치(공정한 발견) |
| 2 | 기업 공개 후에도 **모든 기업의 로고·정보 영역은 동일 규격**으로 표시한다 | 기업 간 시각적 격차 방지 |
| 3 | 아이디어 피드·전광판에서 **기업 정보는 2차 정보**로, 아이디어명·이미지가 먼저 보인다 | 브랜드 인지도에 따른 투표 편향 방지 |
| 4 | 정보를 **색만으로 구분하지 않는다** (아이콘·텍스트 병행) | 접근성 |
| 5 | 터치 영역 최소 44px, 키보드 포커스·스크린리더 레이블 제공 | 접근성 |
| 6 | 개인정보 제공(연락처 등)은 **기본 OFF 옵트인** | 개인정보 원칙 |
| 7 | AI 결과물에는 **면책 문구**와 `AI 활용 콘셉트 제안` 표시를 항상 포함 | 대회 규정·기업 보호 |
| 8 | 대시보드(S02)의 모든 수치에 `가상 예시` 표기 | 데이터 신뢰성 |
| 9 | 비활성 CTA에는 **비활성 이유를 표시**한다 | 이탈 방지 |
| 10 | 정상 화면 외 **로딩·빈 데이터·오류 상태**를 모든 화면에 함께 설계한다 | 행사장 네트워크 환경 대응 |

# 2. 확정 자산: MIXI 마스코트 (제작 완료)

- 블루 톤 3D 로봇 캐릭터, **LED 페이스로 표정이 바뀌는 구조** (눈웃음, 숫자, 화살표 등 픽셀 표정 확장 가능)
- 마스코트의 역할은 **장식이 아니라 기능**: 안내 · 진행상태 표시 · 축하 피드백

| 등장 지점 | 기능적 역할 | 활용 가능한 LED 표정 예 |
|---|---|---|
| P01 시작 | 참여 방법 안내 (환영) | 눈웃음 |
| P03 블라인드 | 탐색 가이드 | 물음표·탐색 표정 |
| P05 토큰 획득 | 획득 축하 피드백 | `1 1` 등 숫자 카운트, 별 |
| P07 AI 생성 | 생성 진행상태 안내 | `> 1` 화살표·로딩 표정 |
| P09 투표 | 순위·축하 | 왕관·하트 등 |

운영 원칙(기능 요구): **마스코트는 핵심 콘텐츠·CTA를 가리지 않아야 한다.** 그 외 비중·크기는 디자이너 재량 — 참고치로 결과 이미지 내 10~15% 이하, 비핵심 화면에서는 축소 사용을 권장(구속 아님).

## P01. 시작·온보딩 `/`

**목적: 5초 안에 차별점을 이해하고 체험을 시작한다.**

- [M] 체험 시작 (익명 세션 생성)
- [M] 기존 이력이 있으면 "이어서 하기" 제공
- [M] 5단계 미니 플로우 안내 (발견→공개→방문→리믹스→공유)
- [M] 예상 체험시간 안내 (약 10~15분)
- [M] 진입 URL 파라미터로 전시 ID 자동 설정 (실물 QR 스캔 진입은 프로토타입 제외 — §3과 동일 기준)
- [S] 닉네임 설정
- [M] 개인정보 최소수집 안내 진입점
- 상태: 정상 / 세션 복구 / 행사 종료 / 네트워크 오류

## P02. 관심 문제·미션 선택 `/mission`

**목적: 기업·제품보다 "해결하고 싶은 생활문제"를 먼저 선택하게 한다.**

- [M] 메인 미션 1개 필수 선택 (미션 카드 6종: 폭염 속 자취방 전기료 줄이기 / 1인 가구 식품 낭비 줄이기 / 플라스틱 없는 뷰티 루틴 / 출퇴근 시간 줄이는 이동 서비스 / 정전에도 작동하는 미래도시 / 직접 입력)
- [M] 직접 입력 40자 제한 + 글자수 카운터
- [M] 관심 분야 1~3개 선택 (환경·에너지 / 일상·생활 / 건강·의료 / 모빌리티 / 식음료 / 미래도시)
- [M] 미션 미선택 시 CTA 비활성 + 이유 표시
- [S] 선택 이유 한 줄 입력
- [S] 선택 완료 시 추천받을 기술카드 수·예상 체험 시간대 안내
- 규칙: 선택값은 기술 추천과 AI 생성 프롬프트에 사용된다. 이 화면에서 기업명·브랜드 요소를 노출하지 않는다.

## P03. 블라인드 기술카드 `/blind-tech`

**목적: 기업 인지도가 아닌 기술의 문제해결력으로 선택하게 한다.**

카드에 표시해야 하는 정보 (이것"만" 노출):
- [M] 기술 번호 (`BLIND TECH CARD No.12 / 20` — 전체 수 함께 표기)
- [M] 이 기술이 해결하는 문제
- [M] 핵심 기능 1문장
- [M] 적용 분야 태그 2~3개
- [M] `기업명 비공개` 배지
- [S] 일상 활용 가능성 표시

기능:
- [M] 카드 목록 무작위 순서 노출 + 전 카드 동일 규격 — **무작위는 최초 진입 시 기본 정렬**이며, 사용자가 명시적으로 정렬을 변경한 경우에만 순서가 바뀐다
- [M] 카드 상세 보기 (해결 문제 / 핵심 기능 / 활용 예시)
- [M] 기술 선택 — 최대 3개, 선택 후 확인 단계
- [M] 선택 현황 표시 — `선택한 기술 n / 3` 카운터 + 선택 카드 요약 (표현 방식은 재량)
- [M] 선택 완료·이미 선택한 기술 상태 표시
- [S] 필터·정렬 (추천순 / 새로운 기술 / 관심분야 / + 검색) — **블라인드 단계에서 '인기순' 정렬은 제공하지 않는다** (초기 선택 쏠림 유발, §1-2 ① 공정성 제약의 연장)
- [S] 선택 이유 기록 (`필요성 / 신기함 / 활용성`)
- [S] 카드별 체류시간 측정 (분석용)
- 규칙: 블라인드 유지를 위해 **선택 시점에는 동일 기업 여부를 알려주지 않는다** — 같은 기업 토큰만 모인 경우의 탈출 경로는 P06에서 [M]으로 제공
- 상태: 로딩 스켈레톤 / 데이터 없음 / 선택 완료 / 이미 선택함

## P04. 기업명·부스 위치 공개 `/reveal/:techId`

**목적: 호기심을 실제 부스 방문으로 전환한다.**

- [M] 선택 직후 기업 공개 연출 (리빌) — 연출 방식은 재량, "가려짐→공개"의 전환이 인지되면 됨
- [M] 표시 정보: 기업명 · 로고(동일 규격) · 정식 기술명 · 쉬운 설명 · 개발 배경 2~3문장 · 부스 위치(홀·번호) · 부스에서 확인할 미션 1개
- [M] 전시장 지도 표시 (정적 이미지 허용) + 길찾기 안내 보기
- [M] 부스 방문 전 토큰 "잠금" 상태 표시
- [S] 예상 이동시간
- [S] 관심 기업 즐겨찾기 · 방문 일정 추가 · 기술 저장
- 상태: 정상 / 지도 정보 없음(텍스트 위치만) / 부스 운영 종료

## P05. 실제 부스 QR 인증 `/verify/:techId`

**목적: 실제 방문을 확인하고 토큰을 활성화한다.**

- [M] QR 스캔 (카메라) — 프로토타입에서는 `부스 인증하기` 버튼으로 대체 가능
- [M] 카메라 권한 거부 시 숫자 인증코드 입력 대체
- [M] QR 값 ↔ 선택 기술 매칭 검증, 잘못된 부스 QR 안내
- [M] 중복 인증 차단 (부스별 고유 QR)
- [M] 인증 성공: 토큰 발급 + 보유 현황(`1 / 3`) + 다음 행동 분기 (다음 기술 찾기 / 토큰함 보기)
- [S] 플래시 켜기 / 카메라 전환 토글 (어두운 전시장 조명 대응)
- [S] 인증 전 체크 안내 (기술 설명 듣기 · 체험하기)
- [S] 직원 도움 요청 진입점 / 네트워크 장애 시 운영요원용 수동 인증
- [S] 성공 직후 1탭 부스 만족도 입력 (선택 사항)
- 상태(전부 필수 설계): 권한 허용 전 / 스캔 중 / 성공 / 잘못된 QR / 중복 인증 / 네트워크 오류

## P06. 나의 기술 토큰 `/wallet`

**목적: 수집 상태를 보여주고 서로 다른 기업의 기술 2~3개를 모으게 한다.**

- [M] 보유 토큰 목록 + 잠긴 슬롯 (총 3슬롯, `보유 2 / 3` 카운터)
- [M] 토큰 정보: 기술명 · 기업명 · 부스 위치 · 핵심 역할 · 기술 카테고리(아이콘+텍스트) · 조합 가능 여부
- [M] 조합 선택: 최소 2개~최대 3개, **한 기업 기술 반복 선택 제한**
- [M] 리믹스 CTA 활성 규칙: 2개 이상일 때만 활성, 미달 시 이유 표시 (`리믹스까지 1개의 기술이 더 필요해요`)
- [S] 조합 미리보기 — 각 기술이 맡을 역할을 조합 전에 보여줌
- [S] 추천 다음 기술 (미션·관심분야 기반)
- [S] 진행률 1/3→2/3→3/3 표시, 서로 다른 기업·분야 조합 배지
- [S] 기업 AI 허용등급에 따른 조합 가능 범위 표시
- [M] 토큰 삭제·교체 — 같은 기업 토큰만 보유해 조합이 불가능한 상태의 탈출 수단
- 상태: 0개 / 1개 / 2개(활성) / 3개(최대) / **조합 제한(같은 기업 토큰만 보유)** — 이 상태에서는 비활성 이유와 함께 `다른 기업의 기술 찾아보기`(P03 이동) + 토큰 교체를 다음 행동으로 제공

## P07. AI 리믹스 설정·생성 `/remix`
